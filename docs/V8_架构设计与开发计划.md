# V8 架构设计：数据库全面升级

编写时间：2026-04-07
基准状态：V7 已完成，codex/v7 分支
目标：大幅扩充数据层，新增技术指标预计算、资金流向、龙虎榜、财务全字段

---

## 1. 背景与目标

V7 闭合了「选股→计划→执行→复盘→模式识别」的核心交易闭环。
V8 的目标是**全面升级数据基础**，为后续的 LLM 联网分析智能化奠定基础。

当前数据库的不足：
- 只有 3 张数据表（stock_basic / daily_quote / stock_financial）
- daily_quote 只有 9 个字段，无技术指标预计算
- stock_financial 只有 2 个字段（profit_dedt / revenue）
- 历史数据仅 3 年
- 无资金流向、龙虎榜等短线核心数据

**V8 核心交付物**：
- 技术指标预计算表（MACD / KDJ / RSI / 布林带 / ATR / OBV / 量比 / MA）
- 个股资金流向表
- 龙虎榜汇总 + 营业部明细表
- 财务指标全字段表（50+ 字段）
- 历史数据扩充至 2014 年
- 分批回填调度器（断点续传）

**V8 不做的事**：
- 不新增 API 端点
- 不修改前端页面
- 不新增筛选条件
- 不修改 LLM Agent
- 不加数据质量监控
- 不换数据库（继续 SQLite）

## 3. 指标计算逻辑

所有指标在 `services/indicator_calc.py` 中实现。
输入：按时间升序排列的行情数据序列。
输出：各指标的数值，数据不足时返回 None。

### 3.1 参数表

| 指标 | 参数 | 类型 | 说明 |
|------|------|------|------|
| MA | 5, 10, 20, 60, 120, 240 | 窗口型 | 简单移动平均 |
| MACD | (12, 26, 9) | 累积型 | 指数移动平均 |
| KDJ | (9, 3, 3) | 累积型 | K/D 初始值 = 50 |
| RSI | 6, 12, 24 | 累积型 | Wilder 平滑 |
| BOLL | (20, 2) | 窗口型 | 标准差通道 |
| ATR | 14 | 累积型 | 真实波幅均值 |
| OBV | - | 累积型 | 累积量能 |
| 量比 | 5日基准 | 窗口型 | 今量/5日均量 |
| 换手率变化 | 5日基准 | 窗口型 | 今换手/5日均换手 |

### 3.2 计算公式

**MA（简单移动平均）**
```
MA_N = sum(close[-N:]) / N
数据不足 N 天时返回 None
```

**MACD (12, 26, 9)**
```
EMA_fast = EMA(close, 12)      首日 EMA = close
EMA_slow = EMA(close, 26)      首日 EMA = close
DIF = EMA_fast - EMA_slow
DEA = EMA(DIF, 9)              首日 DEA = DIF
MACD_HIST = (DIF - DEA) * 2
```

**KDJ (9, 3, 3)**
```
RSV = (close - LOW_9) / (HIGH_9 - LOW_9) * 100
     HIGH_9 = 最近9日最高价, LOW_9 = 最近9日最低价
     HIGH_9 == LOW_9 时 RSV = 50
K = K_prev * 2/3 + RSV * 1/3   初始 K = 50
D = D_prev * 2/3 + K * 1/3     初始 D = 50
J = 3*K - 2*D
```

**RSI (N)**
```
change = close - prev_close
U = max(change, 0)
D = max(-change, 0)
avg_U = EMA(U, N)    Wilder平滑: prev * (N-1)/N + current * 1/N
avg_D = EMA(D, N)
RS = avg_U / avg_D
RSI = 100 - 100 / (1 + RS)
avg_D == 0 时 RSI = 100
```

**布林带 (20, 2)**
```
MID = MA(close, 20)
STD = 标准差(close[-20:])
UPPER = MID + 2 * STD
LOWER = MID - 2 * STD
数据不足 20 天时返回 None
```

**ATR (14)**
```
TR = max(high - low, |high - prev_close|, |low - prev_close|)
ATR = ATR_prev * 13/14 + TR * 1/14     首日 ATR = TR
```

**OBV**
```
if close > prev_close: OBV = OBV_prev + vol
elif close < prev_close: OBV = OBV_prev - vol
else: OBV = OBV_prev
首日 OBV = 0
```

**量比**
```
volume_ratio = today_vol / avg(vol[-5:])
5日均量为0时返回 None
```

**换手率变化**
```
turnover_change = today_turnover_rate / avg(turnover_rate[-5:])
5日均换手为0时返回 None
```

### 3.3 累积型指标的处理

累积型指标（MACD / KDJ / RSI / ATR / OBV）的今日值依赖昨日值：
- **历史回填**：必须按时间正序逐日计算，不能跳日
- **每日增量**：读取该股票前一天的指标值作为初始状态
- **首次计算**：使用默认初始值（如 KDJ 的 K=D=50）

## 4. 同步流程设计

### 4.1 每日增量同步（15:35 触发）

```
sync_daily() 扩展为：
  1. sync_stock_basic()        — 更新股票列表（现有）
  2. sync_daily_quotes()       — 拉最近 5 天行情（现有）
  3. sync_moneyflow_daily()    — 拉最近 5 天资金流向（新增）
  4. sync_toplist_daily()      — 拉最近 5 天龙虎榜（新增）
  5. sync_financial_data()     — 增量拉取全字段财务数据（修改）
  6. sync_indicators_daily()   — 计算最近 5 天指标（新增）
```

顺序严格：行情必须先于指标计算。

### 4.2 历史回填（手动触发，一次性）

```
backfill_all():
  1. backfill_daily_quotes("20140101")       — 约 2750 交易日
  2. backfill_moneyflow("20150101")          — 约 2500 交易日
  3. backfill_toplist("20150101")            — 约 2500 交易日
  4. backfill_financial()                    — 删旧表，全量重拉
  5. backfill_indicators("20150101")         — 纯计算，按股票逐只处理
```

每步设计要点：
- **按交易日分批**：每次拉一天全市场数据，sleep 0.35 秒
- **断点续传**：检查表中已有的最新日期，从下一天开始
- **进度打印**：每 20 天打印一次进度
- **容错**：单日拉取失败打日志跳过，不中断整体
- **财务特殊处理**：按股票逐个拉取，sleep 0.25 秒，每 100 只 commit 一次

### 4.3 速率控制

Tushare 6000 积分权限的频率限制约为每分钟 500 次。
保守设置 sleep 0.35 秒/次（约 170 次/分钟），留足余量。

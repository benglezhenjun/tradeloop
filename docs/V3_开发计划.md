# V3 开发计划：仪表盘 + 市场温度

编写时间：2026-04-05
基准状态：V2 已完成打磨（见 `V2_打磨完成进度书.md`），main 分支为稳定基线

---

## 1. 背景与目标

V2 完成了自选股管理和 K 线图表，用户已能管理关注股票并查看个股详情。

V3 的目标是"市场全局感知"：用户打开应用时，第一眼就能看到**今天市场的整体状态**，而不只是单只股票的数据。

**V3 核心交付物**：
- 一个仪表盘页面（Dashboard），取代当前空洞的首页
- 三个市场温度指标：市场概览 / 行业热度 / 市场宽度趋势

**V3 不做的事**（留到后续版本）：
- LLM 分析（V4）
- 交易计划/持仓（V5-V6）
- 预警推送（V10）
- 个人绩效曲线

---

## 2. 可用数据基础

V3 所有指标均可从现有数据表计算，**不需要新增 Tushare API 调用，不需要新增数据表**。

| 数据表 | 关键字段 | 用途 |
|--------|---------|------|
| `daily_quote` | `ts_code, trade_date, pct_chg, amount, total_mv, vol` | 涨跌计算、成交额汇总 |
| `stock_basic` | `ts_code, name, industry, market` | 行业分组 |

**单位约定（继承 V1/V2）**：
- `amount` 存储单位：千元。转亿元：`amount / 100_000`
- `total_mv` 存储单位：万元。转亿元：`total_mv / 10_000`

## 3. 功能规格

### 3.1 后端：新增服务和路由

#### `backend/app/services/market.py`（新建）

三个函数，均接受 `db: Session` 参数：

**`get_market_overview(db, trade_date: str | None = None) -> dict`**

参数：`trade_date` 为 None 时自动取 `daily_quote` 中的 `MAX(trade_date)`。

返回结构：
```python
{
    "trade_date": "20260405",
    "up_count": 2500,          # 上涨家数（pct_chg > 0）
    "down_count": 1800,        # 下跌家数（pct_chg < 0）
    "flat_count": 200,         # 平盘家数（pct_chg == 0）
    "limit_up_count": 85,      # 涨停家数
    "limit_down_count": 12,    # 跌停家数
    "total_amount_yi": 8500.0, # 全市场成交额（亿元）
    "avg_pct_chg": 0.45,       # 全市场平均涨跌幅（%）
}
```

涨停判断规则：
- `ts_code` 以 `688` 开头（科创板）或 `300`/`301` 开头（创业板）：`pct_chg >= 19.9` 为涨停，`<= -19.9` 为跌停
- 其余（主板）：`pct_chg >= 9.9` 为涨停，`<= -9.9` 为跌停
- 注意：`pct_chg` 为 NULL 的记录（停牌/新股）跳过所有计数

---

**`get_industry_heat(db, trade_date: str | None = None) -> list[dict]`**

按行业聚合当日行情，按平均涨跌幅降序排列。

返回结构（列表，每项一个行业）：
```python
[
    {
        "industry": "电子",
        "stock_count": 320,        # 该行业今日有行情的股票数
        "avg_pct_chg": 2.15,       # 行业平均涨跌幅（%）
        "total_amount_yi": 450.0,  # 行业成交额合计（亿元）
        "up_count": 200,
        "down_count": 110,
    },
    ...
]
```

过滤规则：
- `StockBasic.industry` 为 NULL 或空字符串的股票不计入任何行业
- 行业内无行情数据的行业不出现在结果里

---

**`get_breadth_history(db, days: int = 30) -> list[dict]`**

取最近 N 个有数据的交易日，每天统计上涨/下跌家数。

返回结构（列表，按日期升序）：
```python
[
    {
        "trade_date": "20260305",
        "up_count": 2100,
        "down_count": 2200,
        "flat_count": 180,
    },
    ...
]
```

查询方式：
- 取 `daily_quote` 中 `pct_chg IS NOT NULL` 的数据，按 `trade_date` 分组
- 取最近 `days` 个不同 `trade_date`

#### `backend/app/api/dashboard.py`（新建）

三个端点，均为 `GET`，无必填参数：

```
GET /api/dashboard/overview
GET /api/dashboard/industry_heat
GET /api/dashboard/breadth
```

错误处理：
- 数据库无行情数据时（例如从未同步），返回 200 + 空结构，不返回 500
  - overview 返回全字段为 0/null 的对象，附加 `"trade_date": null`
  - industry_heat 返回空列表 `[]`
  - breadth 返回空列表 `[]`

在 `backend/app/main.py` 中注册路由：
```python
from app.api import dashboard
app.include_router(dashboard.router, prefix="/api/dashboard")
```

```typescript
  trade_date: string | null

  down_count: number

  trade_date: string
  up_count: number
  down_count: number
  flat_count: number

```

#### `frontend/src/stores/dashboard.ts`（新建）

Pinia store，管理三类数据的加载状态：
```typescript
state: { overview, industryHeat, breadth, loading }
actions: { fetchAll() }
```

#### `frontend/src/views/DashboardView.vue`（新建）

页面布局分三区：

**顶部 — 市场概览卡片行**（6 张 el-card）：
- 上涨家数（绿色）
- 下跌家数（红色）
- 涨停家数
- 跌停家数
- 全市场成交额（亿元）
- 市场平均涨跌幅（%）

**中部 — 行业热度横向柱状图**（ECharts）：
- X 轴：平均涨跌幅（%）
- Y 轴：行业名称，按涨跌幅降序排列（涨幅最高在顶部）
- 颜色：正值绿色，负值红色（使用 ECharts `itemStyle` 条件着色）
- 高度：行业数量 × 28px，最低 400px（防止行业过多时太密）

**底部 — 市场宽度趋势图**（ECharts 堆叠面积图）：
- X 轴：交易日期
- Y 轴：家数
- 系列：上涨（绿）/ 下跌（红）/ 平盘（灰）堆叠

**无数据状态**：
- 当 `overview.trade_date === null` 时，显示提示文字："尚无行情数据，请先在设置页触发数据同步"

## 5. 验收标准

1. 仪表盘页面可正常打开，三个区域均有内容（前提：本地已同步行情数据）
2. 无数据时显示空状态提示，不崩溃、不出现 NaN
3. 行业热度图涨幅最高的行业显示在顶部，正负值颜色正确
4. 涨停数量与当日实际情况大致一致（人工抽查 1-2 日）
5. 市场宽度图近 30 日数据曲线平滑显示
6. `pnpm build` 通过，无 TypeScript 错误
7. `pytest tests/ -v` 全部通过（含新增的 dashboard 测试）

## 6. 已知约束与边界

- **涨停判断是近似值**：实际涨停还受新股首日规则、ST股规则影响。当前逻辑以 pct_chg 阈值为准，已能覆盖绝大多数情况，不做特殊处理。
- **行情数据依赖每日同步**：仪表盘展示的是最新同步日的数据，若用户未同步则只能显示空状态。
- **行业分类来源 Tushare**：`stock_basic.industry` 字段来自 Tushare，分类粒度和 A股常见分类有差异，V3 直接用，不做重映射。
- **不做实时刷新**：页面数据在 `onMounted` 时加载一次，不做 WebSocket 或轮询（V3 阶段定位是"今日收盘后复盘工具"，不需要实时）。

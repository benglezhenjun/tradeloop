## 1. 背景与目标

V5 让系统能"做计划"，V6 让系统能"记录执行、跟踪持仓"。
V7 的目标是**闭合核心交易闭环**：计划 → 执行 → 记录 → **复盘 → 模式学习**。

用户清仓或完成一轮交易后，可以生成复盘报告：
- LLM 对比计划与实际执行，生成 8 维度评分
- 用户补充自身反思
- 系统跨多次复盘识别行为模式（优势 / 弱点）

**V7 = 交易复盘 + 8 维度评分 + 行为模式识别**

**V7 核心交付物**：
- 复盘报告 CRUD（生成 / 查看 / 补充反思 / 删除）
- LLM 复盘 Agent（对比计划 vs 实际，生成评分 + 分析）
- 8 维度雷达图可视化
- 行为模式识别（跨复盘汇总，识别正面 / 负面模式）
- 持仓页集成"生成复盘"入口

**V7 不做的事**（留到后续版本）：
- NAV 净值曲线与策略对比（V8）
- 回测引擎（V9）
- 预警通知（V10）

## 2. 数据模型

### 2.1 TradeReview 表（新增）

```python
class TradeReview(Base):
    __tablename__ = "trade_review"

    id              = Column(Integer, primary_key=True, autoincrement=True)
    ts_code         = Column(String(20), nullable=False, index=True)
    stock_name      = Column(String(50), nullable=False)
    plan_id         = Column(Integer, ForeignKey("trading_plan.id", ondelete="SET NULL"), nullable=True)

    # 交易摘要（生成时快照）
    total_buy_amount    = Column(Float, nullable=False)          # 累计买入金额
    total_sell_amount   = Column(Float, nullable=False)          # 累计卖出金额
    total_fee           = Column(Float, nullable=False)          # 累计手续费
    realized_pnl        = Column(Float, nullable=False)          # 已实现盈亏
    trade_count         = Column(Integer, nullable=False)        # 交易笔数
    first_trade_date    = Column(String(10), nullable=False)     # 首笔交易日期
    last_trade_date     = Column(String(10), nullable=False)     # 末笔交易日期
    holding_days        = Column(Integer, nullable=False)        # 持仓天数

    # 8 维度评分
    scores              = Column(Text, nullable=False)           # JSON: {"entry_timing": 7, ...}
    overall_score       = Column(Float, nullable=False)          # 总评分（有效维度均值）

    # LLM 分析
    analysis            = Column(Text, nullable=False)           # Markdown — 综合复盘
    improvement         = Column(Text, nullable=True)            # Markdown — 改进建议

    # 用户反思
    user_notes          = Column(Text, nullable=True)

    # 元数据
    created_at          = Column(String(30), nullable=False)
    updated_at          = Column(String(30), nullable=False)
```

**索引**：
- `ix_review_ts_code` on `ts_code`
- `ix_review_plan_id` on `plan_id`

### 2.2 BehaviorPattern 表（新增）

```python
class BehaviorPattern(Base):
    __tablename__ = "behavior_pattern"

    id              = Column(Integer, primary_key=True, autoincrement=True)
    pattern_type    = Column(String(20), nullable=False)         # "strength" / "weakness"
    title           = Column(String(100), nullable=False)        # 简短标题
    description     = Column(Text, nullable=False)               # 详细描述
    dimension       = Column(String(30), nullable=True)          # 关联维度（可空=综合）
    evidence_ids    = Column(Text, nullable=True)                # JSON 数组：关联的 review ID
    status          = Column(String(20), nullable=False, default="active")  # active / resolved / dismissed
    created_at      = Column(String(30), nullable=False)
    updated_at      = Column(String(30), nullable=False)
```

### 2.3 8 维度定义

| 键名 | 中文名 | 评分范围 | 数据来源 |
|------|--------|---------|---------|
| `entry_timing` | 入场时机 | 1-10 | 入场价 vs MA/支撑位，行情数据 |
| `exit_timing` | 离场时机 | 1-10 | 离场价 vs 后续走势，行情数据 |
| `stop_loss` | 止损纪律 | 1-10 | 计划止损价 vs 实际最大回撤 |
| `take_profit` | 止盈执行 | 1-10 | 计划止盈梯度 vs 实际卖出记录 |
| `position_sizing` | 仓位管理 | 1-10 | 实际仓位 vs 总资金比例 |
| `holding_period` | 持仓周期 | 1-10 | 持仓天数 vs 趋势合理性 |
| `discipline` | 交易纪律 | 1-10 | 操作频率、是否追涨杀跌 |
| `risk_reward` | 盈亏比 | 1-10 | 实际风险回报比 |

- 无关联计划时，`stop_loss` 和 `take_profit` 仍由 LLM 根据实际行为打分（而非"是否遵守计划"）
- `overall_score` = 8 个维度分数的算术平均值
- scores JSON 格式：`{"entry_timing": 7, "exit_timing": 6, ...}`

## 3. 后端服务设计

### 3.1 API 路由

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/review/generate/{ts_code}` | LLM 生成复盘（需要该股有交易记录） |
| `GET` | `/api/review` | 复盘列表（支持 `?ts_code=` 筛选） |
| `GET` | `/api/review/{id}` | 复盘详情 |
| `PUT` | `/api/review/{id}/notes` | 更新用户反思 |
| `DELETE` | `/api/review/{id}` | 删除复盘 |
| `GET` | `/api/review/stats` | 复盘统计（总数、平均分、最佳/最差维度） |
| `GET` | `/api/review/patterns` | 获取行为模式列表 |
| `POST` | `/api/review/patterns/refresh` | LLM 重新分析行为模式 |
| `PATCH` | `/api/review/patterns/{id}/status` | 更新模式状态 |

### 3.2 复盘生成流程（核心）

```
POST /api/review/generate/{ts_code}
```

1. **数据采集**：
   - 查询该股所有 `TradeRecord`（按时间升序）
   - 查询该股 `Position`（持仓快照）
   - 查询关联的 `TradingPlan`（如有多个取最近的 `executed` 计划）
   - 查询持仓期间的 `DailyQuote`（入场日 → 末笔交易日 + 20 天后续）
   - 读取 `total_capital` 配置

2. **构建交易摘要**：
   - 计算：累计买入/卖出金额、总费用、已实现盈亏、交易笔数
   - 计算：持仓天数 = 末笔日期 - 首笔日期
   - 格式化所有交易记录为时间线文本

3. **组装 LLM Prompt**：
   - 交易时间线 + 计划（如有）+ 持仓期间行情走势
   - 要求 LLM 返回 JSON：8 维度评分 + 综合分析 + 改进建议

4. **解析 LLM 响应**：
   - 提取 JSON → 验证 8 个维度分数在 1-10 范围
   - 计算 overall_score
   - 提取 analysis 和 improvement 文本

5. **保存**：
   - 创建 `TradeReview` 记录
   - 返回完整复盘数据

### 3.3 行为模式分析流程

```
POST /api/review/patterns/refresh
```

1. **数据采集**：
   - 查询所有 `TradeReview`（至少需要 3 条才有意义）
   - 统计各维度平均分、标准差、趋势（近期 vs 早期）

2. **组装 LLM Prompt**：
   - 汇总数据 + 各复盘的分数分布
   - 要求 LLM 识别 2-5 个行为模式（优势 + 弱点）
   - 返回 JSON 数组

3. **更新模式**：
   - 清除旧的 `active` 模式
   - 插入新识别的模式
   - 返回模式列表

### 3.4 复盘统计

```
GET /api/review/stats
```

纯数值计算，不调用 LLM：
- `total_reviews`: 复盘总数
- `avg_overall_score`: 平均总评分
- `avg_scores`: 各维度平均分
- `best_dimension`: 最强维度
- `worst_dimension`: 最弱维度
- `total_reviewed_pnl`: 已复盘交易的总盈亏
- `win_count` / `loss_count`: 盈利/亏损笔数

## 6. LLM Prompt 设计

### 6.1 复盘生成 Prompt

```
你是一个 A 股交易复盘助手。根据以下交易数据，对该笔交易进行专业复盘分析。

## 股票信息
- 股票：{stock_name}（{ts_code}）
- 总资金：{total_capital} 元

## 交易计划（如有）
{plan_text 或 "无关联交易计划，为手动交易"}

## 交易记录时间线
{trades_timeline}

## 持仓期间行情走势
{quotes_summary}

## 输出要求
返回 JSON 对象，包含以下字段：

{
  "scores": {
    "entry_timing": <1-10整数>,
    "exit_timing": <1-10整数>,
    "stop_loss": <1-10整数>,
    "take_profit": <1-10整数>,
    "position_sizing": <1-10整数>,
    "holding_period": <1-10整数>,
    "discipline": <1-10整数>,
    "risk_reward": <1-10整数>
  },
  "analysis": "<Markdown 格式综合分析，200-400字>",
  "improvement": "<Markdown 格式改进建议，100-200字>"
}

评分说明：
- 1-3 分：明显不足
- 4-6 分：中等水平
- 7-8 分：表现良好
- 9-10 分：优秀

仅输出 JSON 对象，不要输出其他内容。
```

### 6.2 行为模式分析 Prompt

```
你是一个交易行为分析师。根据以下多次交易复盘的评分数据，识别用户的交易行为模式。

## 复盘数据汇总
- 总复盘数：{count}
- 各维度平均分：{avg_scores}
- 各维度标准差：{std_scores}
- 盈利笔数：{win_count}，亏损笔数：{loss_count}

## 各复盘详情
{reviews_summary}

## 输出要求
识别 2-5 个行为模式，返回 JSON 数组：

[
  {
    "pattern_type": "strength" 或 "weakness",
    "title": "简短标题（10字以内）",
    "description": "详细描述（50-100字）",
    "dimension": "关联维度键名或null",
    "evidence_ids": [关联的复盘ID数组]
  }
]

仅输出 JSON 数组，不要输出其他内容。
```

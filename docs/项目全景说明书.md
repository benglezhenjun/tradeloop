## 一、项目概述

**一句话概括**：一个跑在你自己电脑上的 A 股私人助手网站，数据全部本地存储，AI 分析调用你自己配的大模型。

打开浏览器就能用，完整的使用流程是：

```
看大盘 → 选股 → AI分析 → 做交易计划 → 记录买卖 → 追踪持仓 → AI复盘 → 发现交易毛病
```

**核心特点**：
- 数据完全本地化，不依赖任何云服务（除了拉数据和调 AI）
- AI 模型可自由切换（DeepSeek / Kimi / Ollama / 任何 OpenAI 兼容接口）
- 从数据采集到交易复盘，形成完整闭环

## 二、功能版本演进详解

### V1：基础数据层

> **主题**："先把股票数据搬到你电脑上"

**做了什么**：
- 从 **Tushare**（一个股票数据网站的 API）下载全部 A 股的基本信息
  - 股票代码、名称、行业、所属市场、上市日期、上市状态
- 下载每天的行情数据
  - 开盘价、收盘价、最高价、最低价
  - 成交量（手）、成交额（千元）
  - 涨跌幅（%）、总市值（万元）、换手率（%）
- 全部存到一个本地 SQLite 数据库文件里（`data/stock.db`）

**数据规模**：

| 数据 | 数量 |
|------|------|
| 股票数 | ~5,500 只 |
| 交易日数 | ~2,750 天（2014年至今） |
| 日行情总记录 | **~1,170 万条** |

**通俗理解**：相当于把东方财富的数据搬了一份到你家硬盘上，以后查数据不用上网。

---

### V2：自选股管理

> **主题**："建个文件夹收藏你关注的股票"

**功能**：
- 创建多个**分组**（比如 "科技股"、"医药股"、"短线观察"）
- 在每个分组里添加 / 删除股票
- **搜索股票**（输入代码或名字模糊查找）
- 每只自选股自动显示最新价格、涨跌幅、成交额、市值
- 支持从筛选结果**批量添加**到自选股

**通俗理解**：跟手机炒股软件的自选股列表一样，但可以分很多组。

---

### V3：市场仪表盘

> **主题**："今天大盘怎么样？一眼看完"

**三大板块**：

| 板块 | 内容 |
|------|------|
| **市场概览** | 上涨/下跌/平盘家数、涨停/跌停家数、全市场成交额、平均涨跌幅 |
| **行业热度** | 哪个行业涨得多/跌得多，柱状图排列 |
| **市场宽度** | 最近30天每天涨/跌/平的家数变化趋势，折线图 |

**涨停判断逻辑**（代码里的规则）：

| 板块 | 涨停阈值 | 跌停阈值 |
|------|---------|---------|
| 科创板（688开头）、创业板（300/301开头） | ≥ +19.9% | ≤ -19.9% |
| 其他板块 | ≥ +9.9% | ≤ -9.9% |

**通俗理解**：打开页面就像看今天的"股市天气预报"。

### V4：AI 智能分析

> **主题**："让 AI 帮你写研报"

有两种模式：

#### 模式 1：每日研报

点一下按钮，等 30~90 秒，AI 分 **5 步**帮你分析。每一步是一个独立的 "Agent"（小助手）：

| 步骤 | Agent 名称 | 给 AI 什么数据 | 让 AI 干什么 |
|------|-----------|--------------|------------|
| 1 | 市场分析 | 今日涨跌家数、涨停跌停数、成交额、平均涨跌幅 | 判断市场情绪（偏多/偏空/震荡）、成交量特征、风险信号 |
| 2 | 行业分析 | 涨幅前5 + 跌幅前5 的行业及其数据 | 资金流向哪些板块、弱势板块原因、板块轮动信号 |
| 3 | 自选股分析 | 你自选股最近5天的价格、涨跌、成交额 | 每只股票 1~2 句简评（强势/弱势/放量/缩量） |
| 4 | 筛选结果分析 | 最近一次策略筛选出的股票列表 | 入选股票的共同特征、整体质量、值得关注的个股 |
| 5 | 汇总 | 上面4步的分析结果 | 生成"每日投研日报"（市场总结 + 核心关注 + 明日展望） |

**执行流程**：

```
你点"生成报告"按钮
    ↓
前端发 POST 请求到后端 /api/analysis/daily_report
    ↓
后端按顺序运行 5 个 Agent，每个 Agent：
    1. 从数据库查出需要的数据
    2. 把数据填进预设的提示词模板里
    3. 调用 DeepSeek API
    4. 拿到 AI 回复
    ↓
5 段分析拼成一份完整报告，存进数据库
    ↓
前端展示（Markdown 渲染）
```

#### 模式 2：个股深度分析

选一只股票，等 15~40 秒。

**喂给 AI 的数据**：
- 最近60天走势统计（收盘价、30日涨跌、60日最高/最低、距高低点距离、MA20/MA60、成交额、市值）
- 最近3年年报财务数据（营收、扣非净利润）

**AI 输出**：技术面位置 + 财务基本状况 + 值得关注的点

> 完整的提示词内容见后文 [五、AI 实现细节](#五ai-实现细节)

### V5：AI 交易计划

> **主题**："AI 帮你出三套买入方案"

选一只股票后，AI 自动生成**三套方案**：

| 方案 | 风格标签 | 特点 |
|------|---------|------|
| 激进方案 | `aggressive` | 仓位较大、止损较宽 |
| 稳健方案 | `balanced` | 中等仓位、适中止损 |
| 保守方案 | `conservative` | 小仓位、紧止损 |

**每套方案包含**：

| 字段 | 说明 |
|------|------|
| 入场价 `target_price` | 建议多少钱买入 |
| 止损价 `stop_loss_price` | 跌到多少必须卖掉止损 |
| 分批止盈 `take_profit` | 涨到不同目标位分批卖出（例：涨到X先卖30%，涨到Y再卖50%，涨到Z全卖） |
| 仓位比例 `position_ratio` | 用总资金的多少买（上限40%） |
| 入场理由 `reasoning` | AI 写的文字分析 |
| 风险提示 `risk_comment` | AI 写的风险评估 |

**喂给 AI 的数据**：
- 股票数据：最新价、30日涨跌、60日高低、均线、成交额、市值、财务
- 市场环境：今日涨跌家数、成交额等
- 用户资金：你在设置里填的总资金

**防错机制**：
1. AI 返回的文本可能混杂解释文字 → 代码自动提取其中的 JSON 部分
2. 提取后校验：必须有3套方案、字段完整、止盈比例之和=1.0、仓位≤0.4
3. 如果 AI 格式不对 → 自动降级为"手动创建"模式

你选中一套后可以再手动微调，然后保存为正式计划。

### V6：交易记录 + 持仓管理

> **主题**："记账本 + 仓位自动追踪"

#### 交易记录

记录每一笔买入/卖出：日期、股票、价格、数量、方向。

**自动计算手续费**：

| 费用 | 公式 | 何时收取 |
|------|------|---------|
| 佣金 | 成交额 × 0.025% | 买卖都收 |
| 印花税 | 成交额 × 0.1% | **只有卖出**才收 |
| 过户费 | 成交额 × 0.002% | **只有上海股票**（.SH结尾）才收 |

#### 持仓追踪

自动根据买卖记录计算：
- 当前持仓数量、平均成本
- 浮动盈亏（按最新价算）
- 已实现盈亏（卖出时结算的）
- 分 "持仓中" 和 "已清仓" 两个标签页

**关键业务规则**：

| 规则 | 说明 |
|------|------|
| 卖出限制 | 不能卖超过你持有的数量 |
| 删除交易记录 | 不做逆向回滚，从头重算该股票的持仓 |
| 买入关联计划 | 如果计划还是"待执行"，自动改为"已执行" |
| 卖出与计划 | 卖出不会反向修改计划状态 |

### V7：AI 交易复盘 + 行为模式识别

> **主题**："AI 帮你反思每笔交易，找出你的毛病"

#### 交易复盘

对一只**已清仓**的股票，AI 从 8 个维度打分（每项 1~10 分）：

| 维度 | 代码 | 评价什么 |
|------|------|---------|
| 入场时机 | `entry_timing` | 买入时机好不好 |
| 离场时机 | `exit_timing` | 卖出时机好不好 |
| 止损纪律 | `stop_loss` | 该止损时有没有执行 |
| 止盈执行 | `take_profit` | 该止盈时有没有执行 |
| 仓位管理 | `position_sizing` | 买的量合不合适 |
| 持仓周期 | `holding_period` | 持仓时间长短是否合理 |
| 交易纪律 | `discipline` | 有没有按计划执行 |
| 盈亏比 | `risk_reward` | 赚的和亏的比例 |

**喂给 AI 的数据**：
- 股票基本信息、你的总资金
- 交易计划（如果有）：目标价、止损价、止盈价
- 完整交易时间线：每笔买卖的日期/价格/数量/费用
- 持仓期间的每日行情走势

**前端展示**：用 **雷达图** 显示8个维度得分，一眼看出哪强哪弱。

#### 行为模式识别

当你积累 **3 笔以上复盘**后，AI 会汇总分析你的交易习惯：

**喂给 AI 的数据**：
- 所有复盘的平均分和标准差
- 盈利笔数 vs 亏损笔数
- 每笔复盘的详细分数

**AI 输出**：识别出的行为模式列表，每个包含：

| 字段 | 说明 |
|------|------|
| `pattern_type` | `strength`（优势）或 `weakness`（弱点） |
| `title` | 模式名称 |
| `description` | 详细描述 |
| `dimension` | 对应哪个维度 |
| `evidence_ids` | 依据来自哪几笔复盘 |

**例子**：AI 可能告诉你 —— "你在止损方面一直表现不好，连续3笔交易止损分都很低，总是舍不得割肉"。

### 贯穿所有版本：选股策略引擎

> **主题**："自由搭配条件来筛选股票"

#### 8 种内置筛选条件

| 条件 | 代码 | 类别 | 作用 | 默认值 |
|------|------|------|------|--------|
| 成交额大于 | `amount_gt` | 量价 | 只看今天成交额超过X亿的 | 20 亿 |
| 成交额排名 | `amount_rank` | 排名 | 只看全市场成交额前N名 | 前 100 |
| 市值大于 | `market_cap_gt` | 市值 | 只看市值超过X亿的 | 100 亿 |
| 均线附近 | `ma_proximity` | 技术 | 价格在MA(N)上方但偏离不超过X% | MA20, ≤6% |
| 均线斜率 | `ma_slope` | 技术 | 年线缓慢上行 | MA240, ≤0.2%/天 |
| 均线多头排列 | `multi_ma_alignment` | 技术 | 短期均线在长期均线上方 | 宽松模式 |
| 区间涨幅 | `price_rise_range` | 技术 | 近N天最大涨幅超过X% | 240天, ≥50% |
| 利润增长 | `profit_growth` | 基本面 | 连续N年扣非净利润增长 | 3 年 |

#### 2 套内置策略

**策略1：热点追踪**（所有条件用 AND 连接，全部满足才入选）：

```
成交额 > 2亿
  AND 成交额排全市场前100
  AND 市值 > 10亿
  AND 收盘价在 MA20 附近（上方，偏离 ≤ 6%）
  AND 3年利润增长
```

**策略2：年线回踩**：

```
年线（MA240）斜率舒缓（≤ 0.2%/天）
  AND 近240天涨幅 ≥ 50%
  AND 收盘价在年线附近（偏离 ≤ 6%）
  AND 均线多头排列（宽松模式）
```

#### 策略执行流程

```
你选一个策略 → 点"运行"
    ↓
后端加载当天全部 ~5000 只股票的行情数据
    ↓
按顺序过滤：
  第1个条件 → 剩 ~2000 只
  第2个条件 → 剩 ~500 只
  第3个条件 → 剩 ~200 只
  ... 全部条件过完 → 最终剩 ~30 只
    ↓
结果保存到数据库（可回看历史记录）
    ↓
前端展示结果，可一键加入自选股
```

```
│                                                         │
│  定时任务：每个交易日 15:35 自动同步当天数据                │
│  历史回填：一键拉取 2014 年至今全部历史数据                 │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                       计算层                             │
│                                                         │
│  indicator_calc.py ──→ daily_indicator  1,112 万条       │
│  计算 22 个技术指标：MA/MACD/KDJ/RSI/BOLL/ATR/OBV/量比   │
│                                                         │
│  screening.py ──→ 策略引擎，Pandas 逐条件过滤选股         │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                     AI 分析层                            │
│                                                         │
│  DeepSeek API（通过 OpenAI SDK 调用，可换其他模型）        │
│                                                         │
│  ┌─ 每日研报 Agent  （5步分析 → 一份完整日报）            │
│  ├─ 个股分析 Agent  （技术面 + 基本面综合分析）            │
│  ├─ 交易计划 Agent  （生成激进/稳健/保守3套方案）          │
│  ├─ 交易复盘 Agent  （8维度打分 + 文字分析）              │

┌─────────────────────────────────────────────────────────┐
│                    交易管理层                             │
│                                                         │
│  trading_plan     → 交易计划（AI生成 或 手动创建）        │
│  trade_record     → 交易记录（买入/卖出/费用自动算）      │
│  position         → 持仓状态（根据交易记录自动计算）       │

│  behavior_pattern → 行为模式（AI 汇总识别）              │
└─────────────────────────────────────────────────────────┘
```

### AI 安全机制

| 机制 | 说明 |
|------|------|
| **禁止预测** | 所有系统提示词都明确要求"不预测股价涨跌，不给出买卖建议" |
| **故障降级** | 任何 AI 调用失败时返回 `⚠️ 该部分分析生成失败：...`，不影响其他步骤 |
| **JSON 提取** | 能从 AI 回复中自动提取 JSON（即使 AI 混杂了解释文字） |
| **格式修复** | 复盘 Agent 如果 JSON 格式错误，会自动重新请求 AI 修复 |
| **计划降级** | 如果 AI 的计划格式校验失败，自动降级为手动创建模式 |
| **温度 0.3** | 低温度降低 AI 的"创造性"，让分析更稳定 |

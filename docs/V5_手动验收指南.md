# V5 手动验收指南

本指南面向非技术人员，按顺序操作即可完成 V5 全部功能验收。

---

## 准备工作：启动系统

打开**两个**终端窗口（在 VS Code 底部点 `+` 号新建终端），分别执行：

**终端 1 — 启动后端：**
```
cd D:\构想\stock-assistant\backend
uv run uvicorn app.main:app --reload --port 8000
```
看到 `Uvicorn running on http://127.0.0.1:8000` 就说明启动成功。

**终端 2 — 启动前端：**
```
cd D:\构想\stock-assistant\frontend
pnpm dev
```
看到 `Local: http://localhost:5173/` 就说明启动成功。

然后用浏览器打开 **http://localhost:5173/**。

---

## 验收项 1：总资金设置

**目标：** 确认总资金能保存并持久化。

1. 点击左侧导航栏的 **"设置"**
2. 找到"总资金"输入框，输入 `500000`（50 万）
3. 点击 **保存**
4. 看到提示"保存成功"（或类似成功提示）
5. **刷新浏览器页面**（按 F5）
6. 再次进入设置页，确认总资金仍然显示 `500000`

> **通过标准：** 刷新后数值没有丢失。

---

## 验收项 2：手动创建交易计划

**目标：** 确认可以手动创建一个交易计划。

1. 点击左侧导航栏的 **"交易计划"**
2. 点击页面上的 **"新建计划"** 按钮
3. 填写以下内容：
   - 搜索并选择一只股票（比如输入"平安"搜索）
   - 方向：选 **买入**
   - 目标价：输入 `12.50`
   - 止损价：输入 `11.80`
   - 止盈档位：保持默认或手动添加（所有档位的比例加起来要等于 100%）
   - 仓位比例：拖动滑块，注意**不能超过 40%**
   - 理由：随便填几个字
4. 点击 **保存**
5. 看到提示"计划已创建"，页面回到计划列表
6. 确认列表中出现了刚才创建的计划，状态显示 **待执行**

> **通过标准：** 计划成功创建，列表中可见，状态为"待执行"。

---

## 验收项 3：编辑待执行计划

**目标：** 确认"待执行"状态的计划可以编辑。

1. 在交易计划列表中，找到刚才创建的计划（状态为"待执行"）
2. 点击 **编辑** 按钮
3. 修改目标价为 `13.00`
4. 点击 **保存**
5. 看到提示"计划已更新"
6. 确认列表中目标价已变为 `13.00`

> **通过标准：** 编辑后数据正确更新。

---

## 验收项 4：状态流转（待执行 → 已执行）

**目标：** 确认状态只能往前走，不能退回。

1. 在计划列表中，找到上面那个"待执行"的计划
2. 点击 **执行** 按钮（绿色文字）
3. 在弹出的确认框中点击"确定"
4. 确认状态变为 **已执行**
5. 确认此时**没有**编辑按钮和执行按钮了（已执行的计划不能再改）

> **通过标准：** 状态变为"已执行"，编辑入口消失。

---

## 验收项 5：仓位比例上限 40%

**目标：** 确认前端和后端都阻止超过 40% 的仓位。

1. 新建一个计划
2. 尝试把仓位比例滑块拖到超过 40% 的位置
3. 确认**滑块被限制在 40%**，无法再往右拖

> **通过标准：** 仓位比例无法超过 40%。

---

## 验收项 6：LLM 生成交易计划（三套方案）

**前提：** 需要已配置 LLM（设置页的 LLM 供应商和 API Key 已填好）。如果没配 LLM，跳到验收项 8。

1. 进入任意一只股票的 **详情页**（比如从自选股列表点进去）
2. 点击 **"生成交易计划"** 按钮
3. 等待 LLM 返回（可能需要几秒到十几秒，页面会显示 loading）
4. 返回后应看到 **三栏并排展示**，分别是"激进"、"稳健"、"保守"三套方案
5. 每套方案显示：方向、目标价、止损价、止盈档位、仓位比例、理由
6. 选择其中一套方案（比如点击"稳健"方案下面的 **选择** 按钮）
7. 进入编辑页面，确认所有字段都可以修改
8. 修改任意一个字段（比如改一下目标价），然后点 **保存**
9. 确认跳转到计划列表，新计划状态为 **待执行**

> **通过标准：** 三套方案正常展示，选择后可编辑，保存成功。

## 验收项 7：LLM 返回异常时的降级处理

**说明：** 这个是测试 LLM 出错时系统不会崩溃。正常使用中不一定能复现，但如果在验收项 6 中遇到了以下情况，说明降级机制正常工作：

- 页面显示黄色/橙色警告，提示"LLM 返回异常，请手动创建"
- 自动进入手动编辑页面，股票信息已预填

> **通过标准：** 没有白屏或报错，有友好提示。

---

## 验收项 8：未设置总资金时的提示

**目标：** 确认没有总资金时不能生成计划。

1. 先去 **设置页**，把总资金改为 `0`，保存
2. 进入任意股票详情页，点击 **"生成交易计划"**
3. 应该看到提示 **"请先在设置页设置总资金"**，并引导你去设置页
4. 去设置页把总资金改回 `500000`，保存

> **通过标准：** 总资金为 0 时有明确提示，不会直接报错。

---

## 验收项 9：状态流转 — 放弃计划

**目标：** 确认计划可以标记为"已放弃"。

1. 手动创建一个新计划（参照验收项 2）
2. 在计划列表中点击 **放弃** 按钮（橙色文字）
3. 在弹出的确认框中点击"确定"
4. 确认状态变为 **已放弃**
5. 确认编辑入口消失

> **通过标准：** 状态正确变为"已放弃"。

---

## 验收项 10：计划详情查看

**目标：** 确认可以查看计划的完整信息。

1. 在计划列表中点击任意一个计划
2. 应弹出详情面板（或进入详情页）
3. 确认展示：股票名称、方向、目标价、止损价、止盈档位、仓位比例、理由、创建时间
4. 如果是 LLM 生成的计划，确认可以看到**备选方案**（可能是折叠状态，点击展开）

> **通过标准：** 所有字段完整展示。

# V6 验收与回归指南

## 目标

确认 V6 的交易记录与持仓能力已经真正接通，同时不破坏 V5 既有功能。

## 命令验证

### 后端

```bash
cd backend
uv run pytest tests/ -v
```

验收点：
- 交易记录新增测试通过
- 持仓新增测试通过
- V1-V5 旧测试全部通过

### 前端

```bash
cd frontend
pnpm type-check
pnpm build
```

验收点：
- 新增路由、页面、store、类型定义没有 TypeScript 错误
- 生产构建可完成

## 手工 Smoke 顺序

1. 打开设置页，保存总资金与三项交易费率。
2. 打开个股详情页，生成交易计划。
3. 在交易计划页保存计划，并从计划页点击“录入交易”。
4. 在交易记录页确认预填内容，保存一笔买入。
5. 打开持仓页，确认当前持仓、市值、浮盈亏和交易时间线正常。
6. 从持仓页点击“卖出”，跳转交易记录页并完成卖出录入。
7. 返回交易记录页删除一笔记录，确认持仓自动重算。
8. 回归旧流程：
   - watchlist
   - screening
   - analysis
   - plan generate / plan CRUD

## V6 关键验收项

- 可以手动录入交易。
- 可以从计划页预填交易。
- 可以从持仓页预填卖出。
- 交易费用默认按配置实时计算。
- 可以手动覆盖最终费用。
- 可以查看交易记录详情并删除交易。
- 可以查看当前持仓和已清仓持仓。
- 可以查看持仓详情中的交易时间线。
- 删除交易后持仓会重算。

## 风险检查

- 删除旧交易后，若历史序列出现超卖，后端应返回明确错误，不得静默吞掉。
- 计划状态自动流转只应发生在“买入且关联计划且计划为 pending”的场景。
- 前端交易预填只是辅助，不应替代后端校验。

## 手工 Smoke 顺序

> **前置条件**：系统中需要有至少一只已清仓的持仓（或先手动走一遍 V6 的买入→卖出流程）

### 第一轮：复盘核心流程

1. **创建测试数据**（如果没有已清仓持仓）：
   - 打开交易记录页，录入一笔买入交易（任意股票）
   - 再录入一笔卖出交易（同股票，全部卖出）
   - 确认持仓页该股显示为"已清仓"

2. **生成复盘**：
   - 打开持仓页，切换到"已清仓"Tab
   - 在已清仓的持仓卡片上找到"生成复盘"按钮
   - 点击生成复盘
   - 确认：
     - [ ] Loading 状态正确显示
     - [ ] 生成成功后跳转到复盘页面
     - [ ] 新复盘出现在列表中

3. **查看复盘详情**：
   - 在复盘列表中点击"查看详情"
   - 确认：
     - [ ] 8 维度雷达图正常渲染（无空白或报错）
     - [ ] 各维度分数显示正确（1-10 范围）
     - [ ] LLM 综合分析文本正常显示
     - [ ] 改进建议正常显示
     - [ ] 交易摘要信息（盈亏、笔数、持仓天数）正确

4. **编辑用户反思**：
   - 在复盘详情对话框中填写反思文字
   - 点击保存
   - 关闭对话框后重新打开
   - 确认：
     - [ ] 反思文字已保存并正确显示

5. **复盘统计**：
   - 查看复盘页顶部统计卡片
   - 确认：
     - [ ] 复盘总数正确
     - [ ] 平均评分显示
     - [ ] 最强/最弱维度显示

6. **删除复盘**：
   - 点击某条复盘的删除按钮
   - 确认弹框
   - 确认：
     - [ ] 复盘从列表中消失
     - [ ] 统计数据已更新

### 第二轮：行为模式分析

> **前置条件**：需要至少 3 条复盘记录。重复第一轮步骤 1-2 生成更多复盘。

7. **刷新行为模式**：
   - 切换到"行为模式"Tab
   - 点击"刷新行为模式分析"按钮
   - 确认：
     - [ ] Loading 状态正确
     - [ ] 生成完成后显示模式卡片列表
     - [ ] 每个模式有标题、类型标签（优势/弱点）、描述

8. **管理模式状态**：
   - 在某个模式卡片上点击"标记已解决"
   - 确认：
     - [ ] 模式状态更新为"已解决"
     - [ ] 卡片样式/标签正确反映新状态

### 第三轮：集成验证

9. **持仓页集成**：
   - 打开持仓页
   - 确认已清仓持仓：
     - [ ] 已有复盘的显示总评分 + "查看复盘"链接
     - [ ] 无复盘的显示"生成复盘"按钮
   - 确认当前持仓（holding）不显示复盘按钮

10. **导航**：
    - 确认侧边栏有"交易复盘"导航项
    - 点击导航跳转正确

### 第四轮：V1-V6 回归

11. **旧功能回归**：
    - [ ] 仪表盘正常加载
    - [ ] 自选股列表正常
    - [ ] 筛选功能正常
    - [ ] LLM 分析正常
    - [ ] 交易计划 CRUD 正常
    - [ ] 交易记录 CRUD 正常

## 风险检查

- 无交易记录的股票不应允许生成复盘（后端应返回明确错误）
- LLM 返回格式异常时应降级为错误提示，不应崩溃
- 复盘评分必须在 1-10 范围内，超范围时后端应拒绝
- 行为模式分析不足 3 条复盘时应返回明确提示，不应调用 LLM
- 删除复盘不应影响原始交易记录和持仓数据
- 前端雷达图在分数为 0 或极端值时不应渲染异常

## API 快速验证脚本

如需跳过前端直接验证 API，可用以下命令：

```bash
# 健康检查
curl http://localhost:8000/api/health

# 生成复盘（替换 ts_code）
curl -X POST http://localhost:8000/api/review/generate/000001.SZ

# 复盘列表
curl http://localhost:8000/api/review

# 复盘统计
curl http://localhost:8000/api/review/stats

# 更新反思
curl -X PUT http://localhost:8000/api/review/1/notes \
  -H "Content-Type: application/json" \
  -d '{"notes": "测试反思内容"}'

# 行为模式
curl http://localhost:8000/api/review/patterns

# 刷新模式
curl -X POST http://localhost:8000/api/review/patterns/refresh
```
